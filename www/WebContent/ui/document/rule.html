约定

一、元素定义

框架中包含各种元素，这些元素可大可小，可简单可复杂，甚至还有一些专门用于某些场景使用的元素，为了学习和使用本框架更加方便，我们对预算的定义分为以下四类：

  • 控件：由一个或者一些网页元素组成的单元，例如：文本框、复选框、下拉框。
  • 组件：由一组控件组成的单元，例如：表单组件、表格组件、树组件。
  • 方法：用于完成控件或者组件功能的JavaScript代码，例如：托拽方法、缩放方法、裁剪方法。
  • 插件：区别于框架组件的第三方组件，例如：日期选择插件、富文本插件、图表插件。
  • 视图：特定业务场景下的通用网页结构，例如：评论视图、通讯录视图、购物车视图。

样式的渲染和方法的运行使得元素有不同的类型、状态和参数，定义如下：

  • 类型：同一种界面单元的不同类型，例如：按钮有主要按钮、次要按钮及危险按钮等，导航菜单类型有普通导航及顶部固定导航等。任何时候都应该只为同一个界面元素指定一种类型，默认类型无需指定。。
  • 状态：同一种类型元素允许在不同的状态中切换，例如：按钮有正常状态和不可用状态，下拉菜单有展开及收拢状态，菜单项目有选中和未选中状态等。在同一个界面元素中允许叠加多个状态。。
  • 参数：参数为元素如何展现提供依据，例如：表格是否隔行变色，是否启用鼠标悬停等。可以同时指定多个参数。
二、状态样式

很多状态和参数在不同的控件或组件中具有一致的用法和相同的含义，这些通用的状态和参数具有固定的CSS类名。

名称说明
.active指示控件或者其子项目已经被激活。
.hover指示控件正处于鼠标悬停状态，在CSS中相对于:hover选择器。
.focus指示控件正处于聚焦状态，在CSS中相对于:focus选择器。
.disabled指示控件处于不可用状态，通常与DOM属性disabled="disabled"一起使用。
.collapse指示当前组件可以被折叠或展开。与状态.in一起出现时表示组件已被展开，没有此状态则反之。
.hide指示当前组件被影藏不可见。
.bordered使得组件具备边框样式。
.borderless移除控件的边框样式。
.inverse指示组件使用与默认颜色样式相对的样式。
.fixed指示当前组件位置被固定，不受页面交互影响。

三、名称规则

这里的名称只涉及到前端的名称规则，后端的暂时不在这里展示。

  • 文件编码统一使用 UTF-8 编码；
  • 命名时以符合语义为主要参考指标，CSS属性书写规范，采用统一风格及命名方法；
  • 结构清晰，层级关系明朗，以不超过三级为标准；
  • 同一表现形式的样式要求可重复利用，模块组件类的样式要求可整体外部移植；
  • 编写 CSS 的时候，应当避免使用CSS Hack，能不用则不用;
  • 所有页面默认都针对Firefox 等最接近标准的浏览器进行设计，然后使用 IE 特有条件注释功能进行针对性修正;
  • 请使用英文进行命名，尽量避免使用拼音。命名要求具有可读性，尽量避免使用缩写。命名虽然允许数字，但应尽量避免使用数字命名;
名称说明
ID驼峰命名法treeNode、下划线命名法tree_node
类中线命名法.x-tree-node框架样式类名必须前缀"x-"
方法驼峰命名法nodeClick、连写命名法nodeclick
变量驼峰命名法userName、下划线命名法user_name、连写命名法username
文件连写命名法orglogo、下划线命名法org_logo
控件小写xtree框架控件名必须前缀"x"

四、代码规约

这里指的是自研控件的一些约定。

名称说明
ID所有前端控件都再jQuery对象上开发；
类所有控件的暴露方法都挂载在控件对应的jQuery对象上，方便索引、查询；
方法所有控件都封装在一个闭包下，内部方法都统一在一个方法对象上，方便折起、拎出来；
变量所有方法都要做参数校验和信息提示；
文件元素做参数时都用jQuery对象传，参数名用"$"+标签名，例如："$input"、"$span"
